Versão 0.1.0#

A história do usuário#

Você gosta de ler? Se sua resposta é sim, talvez já tenha tido a experiência de comprar mais livros do que consegue terminar, ou já tenha ficado ansioso diante das 27 abas abertas no navegador enquanto pesquisava algum assunto interessante. Essas e todas as outras infinitas possibilidades que encontramos me fizeram pensar: eu sendo um leitor, quero organizar melhor meu processo de leitura para que eu possa me manter engajado.

A persona#

Meu nome é Francisco. Trabalho com desenvolvimento de software, mas formalmente frequentei o curso de Ciências Sociais. Às vezes sou autodidata, antes da universidade alimentei uma paixão pela literatura que de certa maneira foi ampliada pelos livros e artigos acadêmicos. Para trabalhar preciso continuamente consultar documentações ou textos técnicos em geral. Cada formato traz consigo um determinado tempo de leitura e absorção, além de serem consumidos de maneira física ou digital. Bem, enquanto eu estiver vivo, haverá louça... Enquanto eu estiver vivo, também haverá livros... Além de lavar, eu preciso ler!

certo, louça e leitura infinita.

Testando a organização como funcionalidade#

Usando o Airtable como banco de dados

Meu foco nesse estágio do paginômetro é validar a ideia a partir de um protótipo, entregando valor desde o inicio do desenvolvimento. Então não vou reinventar a roda, posso armazenar pequenas quantidades de texto em qualquer lugar. Escolhi o Airtable. Nele eu consigo criar tabelas para simular um banco de dados e criar formulários para alimentá-lo, simulando o frontend. Além de consumir toda informação a partir de uma API bem documentada.

Testando o protótipo

1. Armazenar textos interessantes#

formulários do Airtable

Olho a mesa do computador e tenho um mini infarto. Eu sendo um leitor tão disperso, preciso centralizar todo esse conteúdo para que eu possa no mínimo encontrar o que tenho que ler! Ou seja, dado que, me interesso por um texto quando o encontro e tenho seus dados, então preciso salvar essa informação em algum lugar. A pilha de interesses.

a própria bolsa da Hermione

2. Classificar textos dentro de um projeto#

uma leve organização

Tenho uma pilha gigante no canto do quarto. Que sensação boa, é quase como chegar em casa depois de uma feira de livros. Mas ainda não é o suficiente. Eu sendo um leitor com interesses tão diversos, preciso classificar minhas leituras para que eu possa acompanhar minha evolução dentro de um determinado tema. Ou seja, dado que, tenho uma lista de textos e alguns podem ser classificados dentro de um determinado assunto, então preciso salvar essa informação em algum lugar. Criando projetos de leitura:

Quantas páginas consigo ler em uma hora?

Com o objetivo de ajudar na organização e entender os meios para proporcionar o desejado engajamento, vou implementar algumas métricas.

40 páginas por hora? Será?

3. Acompanhar a leitura#

começando

Ler e colher dados, chegou o momento desejado! Eu sendo um leitor curioso, gostaria de medir meu processo de leitura para que eu possa entender como realizar isso da melhor forma. Ou seja, dado que, reservo um tempo para leitura e colho dados sobre isso, então preciso armazenar essa informação para rastrear o progresso. Criando o paginômetro:

Espera ai...

O primeiro Sprint#

Esse é o momento de usar o protótipo por alguns dias e finalizar o primeiro projeto de leitura. Tenho apenas duas tarefas, seguir minha rotina com base nas horas estimadas para conclusão do projeto e ao término de cada leitura, seja ela física ou digital, abrir o formulário do paginômetro e adicionar as páginas lidas. Posso fazer isso tanto no computador quanto no celular.

Mas espera ai.Se você é um leitor atento, deve ter se perguntado; como posso saber quanto tempo foi gasto se forneço apenas as páginas lidas? Bem, de inicio eu não quero registrar o tempo de leitura a cada nova entrada. Quero minimizar o custo da execução enquanto não entendo a maneira mais simples de obter as métricas... Essa será uma funcionalidade futura. Nesse primeiro sprint o propósito é validar a facilidade/comodidade de uso e repensar uma série de detalhes, enquanto aprendo mais sobre JavaScript :P

Vamos lá..

Métricas, o próximo passo#

O valor inicial foi entregue, mas o desenvolvimento continua. O próximo passo é consumir a API do Airtable, armazenar os dados num cache intermediário, obter as possíveis e desejadas métricas. O passo seguinte deve ser fácil de imaginar, entre outros, visualizar os dados obtidos!
